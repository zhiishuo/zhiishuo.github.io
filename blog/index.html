笔记与备忘

我这几篇文章其实都在追同一个问题：当多模态系统面对证据退化、模态冲突和高不确定性时，怎样还能做出可靠判断。

2026-04 Workflow

科研流程不是附属品，而是研究本身的一部分

我真正开始认真对待流程，不是因为想把研究做得更规范，而是因为几个项目都在同一个地方出过问题：问题定义、失败样本和执行计划散在不同地方，最后方法越做越漂亮，研究问题却越做越偏。

问题场景

像 AV-RISE、HEME 这样的项目，一旦没有把复杂样本、预期主张和执行顺序固定下来，很容易从“我要解释什么失败”滑向“我还能再加什么模块”。

我的判断

我后来只保留四个外化对象：source of truth、review note、claim/evidence matrix、execution plan。它们的作用不是做文档，而是防止研究在中途换题。

我怎么验证

如果这套流程是有效的，我应该随时能回答三件事：我要解释哪类失败、我准备用什么证据支撑、下一步实验为什么值得做。

我现在更相信这一点：没有外化对象的研究，最后通常会被方法形式带偏；有了清晰的外部骨架，判断才有机会被持续修正。

2025-09 Paper

AV-RISE 让我把重点从融合转向证据可靠性

这条工作最打动我的地方，不是它证明“多模态有用”，而是它逼我承认：在真实音视频语音里，先要解决的根本不是融合，而是证据到底还靠不靠谱。

失败场景

真正难的样本通常不是模态缺一个，而是复杂噪声、轻微不同步、视觉遮挡、口型模糊和弱信号同时出现。模型错的原因，经常不是“信息太少”，而是把不可靠信息也当成了证据。

核心判断

AV-RISE 让我更相信，跨模态去噪应该被看成表征学习本身的一部分。重点不是再加一层更强融合，而是把 multi-level evidence 重新组织起来，做 weak-signal repair 和 progressive enhancement。

迁移价值

后来我转回多模态情感理解时，先问的就不是“这几个模态怎么融合”，而是“这条样本里每个模态到底提供了什么证据”。这不是领域类比，而是同一种 evidence reorganization 问题的迁移。

所以对我来说，AV-RISE 不只是一个鲁棒 AVSR 工作，更像一个判断起点：当证据退化时，系统要先学会分辨哪些信号还能信，再谈后面的融合和决策。

2026-03 Agent

情感 Agent 的核心不是人格，而是介入策略

我不太想把情感 Agent 写成一个“很会共情”的人格壳子。真正值得研究的，是系统在信息不完整的时候，怎么判断该不该说、该先问什么、以及该不该停。

典型失误

很多系统的问题不是不会安慰，而是太快理解、太快安慰、太快给建议。用户状态本来就模糊，它却直接给出确定性的介入，这一步一错，后面很容易越聊越偏。

研究对象

按 CARES 这条线去看，情感支持更像一个 intervention policy：先用 belief tracker 估计 emotion、need、preference、memory、risk、conflict 和 uncertainty，再由 clarification gate 和 stance planner 决定 reflect、clarify、suggest，还是 boundary / abstain。

怎么评估

这件事不该只看“回复像不像人”，而要看 clarification precision / recall、wrong-direct-answer rate、boundary failure、dependence risk，以及版本升级后有没有 destructive forgetting。

所以我现在想做的情感 Agent，不是开放式陪伴人格，而是一个 bounded intervention system：它知道自己什么时候理解得不够，也知道什么时候应该先问、先停，或者干脆不往前走。

2026-04 Models

情感模型下一步，不只是更大，而是会停、会修、会放弃

近两年这条线已经有了强 generalist baseline，也有了更完整的 benchmark。对我来说，真正值得继续挖的，不是统一模型还能不能更大，而是默认答案在 hard case 上什么时候该被怀疑。

失败样本

最值得看的不是平均样本，而是高冲突样本：文本平静，语气紧张，视觉线索又往另一个方向拉。系统常常不是完全没看到证据，而是太快形成了一个顺手的默认答案。

我的判断

AffectGPT、Emotion-LLaMAv2 / MMEVerse 把强 baseline 往前推了很多，但 EMOE、CA-MER / MoSEAR 也已经说明，冲突与不确定不是边角现象。真正的瓶颈更像 selective repair：什么时候该相信默认答案，什么时候该重新组织证据。

我要验证什么

我关心的不是平均分会不会涨，而是 default answer 何时不可靠、只在 hard cases 干预会不会更稳，以及 late corrective re-entry 是否真的优于只改 output head。

所以这篇的中心判断很简单：情感模型下一步的重点，不只是把 unified path 再做大，而是让系统学会识别默认答案的失效边界，并在必要时有节制地修复自己。
